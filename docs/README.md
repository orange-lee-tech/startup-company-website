# docs 文档目录说明

本目录用于存放官网项目的交接文档、部署说明、状态记录和后续维护计划。

## 目录结构

- `deployment/`：服务器部署、Nginx、Nginx Proxy Manager、发布流程、回滚方式等技术运维说明。
- `project/`：官网建设状态、内容口径、阶段性汇报、后续待办等项目层说明。

## 使用建议

1. 新 AI 或维护者接手时，先读根目录 `README.md` 和 `AGENTS.md`，再读本目录。
2. 涉及服务器、域名、HTTPS、缓存、安全头、发布回滚的问题，优先看 `deployment/`。
3. 涉及当前建设状态、已完成事项、待办事项、内容和合规口径的问题，优先看 `project/`。
4. 文档应记录“为什么这样做”，而不是只记录命令。
5. 不要在文档中写入敏感凭据或服务器私密信息。

## 当前重点

当前官网已完成上线、ICP备案展示、HTTPS、主域规范化、搜索引擎基础文件、内容合规降敏、联系页腾讯问卷、服务器缓存与基础安全头配置。

后续重点包括：

- 公安备案号下发后补页脚。
- 增加 JSON-LD 结构化数据。
- 搜索引擎后台复检 sitemap。
- 建立师资和案例授权脱敏台账。
- 持续根据业务反馈优化页面内容和视觉重点。
