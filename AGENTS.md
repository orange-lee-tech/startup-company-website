# AI 协作与仓库接手指南

本文件用于帮助后续 AI、Codex 或真人维护者快速理解九辰教育官网的目标、风格、技术关系和修改边界。

## 1. 项目定位

- 官网地址：`https://jiuchenedu.com/`
- 展示品牌：九辰本硕博升学就业
- 公司主体：长沙九辰教育咨询有限公司
- 官网性质：教育咨询类企业官网，用于品牌展示、服务说明、案例展示、师资展示、常见问题与咨询转化。
- 主域策略：非 www 为唯一主域，即 `https://jiuchenedu.com/`。

本项目不是论坛、后台系统、在线课程系统或动态 CRM。当前以静态官网为主，联系转化通过腾讯问卷承接。

## 2. 当前技术结构

源码仓库使用：

- Next.js
- React
- TypeScript
- Tailwind CSS
- 静态导出产物 `out/`

线上部署链路为：

```text
用户访问
→ Nginx Proxy Manager，处理 80/443、HTTPS、证书、域名转发
→ 宿主机 Nginx 8088
→ /usr/share/nginx/html/jiuchen-site
```

GitHub 仓库只保存源码和静态资源，不保存服务器私有配置、敏感凭据或服务器备份包。

## 3. 内容风格

整体风格应保持：

- 专业、理性、可信赖
- 教育咨询行业的稳健表达
- 避免过度营销、夸张承诺和强刺激销售话术
- 强调规划、评估、陪跑、材料优化、面试训练、路径复盘
- 不把个别案例结果包装成普遍承诺

页面视觉偏好：

- 高端、克制、清晰
- 白底或浅色卡片优先，避免大面积暗沉
- 深色背景可用于品牌氛围区，但案例、表单、信息阅读区应尽量明亮
- 重点信息应通过标题、结果卡片、按钮和留白凸显，而不是堆叠标签

## 4. 合规边界

公开页面应避免使用以下绝对化或夸大表达：

- 保录取
- 包 offer
- 百分百上岸
- 包过
- 必过
- 低分必录
- 对单一学校、导师、公司或岗位作确定性结果承诺

FAQ 或合规说明中可以出现相关词汇，但语义必须是明确否定和风险提示，不应作为营销卖点。

案例、截图、录取材料、聊天记录、师资头像等公开前必须确认授权与脱敏。

## 5. 案例页当前设计口径

根据业务反馈，案例页不再突出“案例有多少个”，避免数量少时形成展示劣势。

当前案例展示重点为：

- 初始情况
- 最终结果
- 代表性路径

不再在前台大量堆叠：

- 学员背景
- 服务过程
- 多个标签
- 当前分类案例数量

案例原始信息仍可保留在 Markdown 数据源中，前台只展示更聚焦的内容。

## 7. 目录说明文件原则

本仓库需要 README 的位置主要是：

- 根目录：说明项目全局状态与维护入口
- `docs/`：说明文档体系
- `public/`：说明公开静态资源
- `public/images/` 及关键图片子目录：说明图片资产用途和授权边界
- `jiuchen-case-markdown/`：说明案例数据源关系
- `src/data/`：说明内容数据层关系

不建议给每个页面源码目录都滥加 README。AI 可以直接读代码，README 应用于说明二进制资产、内容拓扑、维护边界和人工习惯。

## 8. 修改工作流

推荐流程：

1. 先读 `README.md`、`AGENTS.md`、相关目录 README。
2. 明确修改目标：内容、视觉、SEO、服务器配置或资产管理。
3. 本地修改源码。
4. 执行构建：`npm run build`。
5. 预览或检查 `out/`。
6. 发布到服务器。
7. 线上 curl 验证。
8. Git commit / push。

涉及服务器配置时必须先备份，并执行配置测试。

## 9. 不要做的事

- 不要提交敏感凭据、服务器备份包或未脱敏材料。
- 不要把公安备案号提前写入网站，必须等正式下发。
- 不要随意更改主域策略，当前主域是非 www。
- 不要直接删除图片资产，先全文搜索引用路径。
- 不要在未测试腾讯问卷 iframe 的情况下启用严格 CSP。
- 不要把案例数量作为主要卖点。

## 10. 适合交给 Codex 的任务

适合 Codex：

- 多文件重构
- UI 组件系统性调整
- 批量迁移案例数据
- 增加 JSON-LD 结构化数据并跑构建检查
- 清理旧模板页源码

适合普通 AI 对话：

- 文案审查
- 方案判断
- 命令指导
- 合规风险提示
- 交接总结
